# 2. 자바의 역사와 제임스 고스링

## 자바의 아버지, 제임스 고스링 (James Gosling)
자바(Java)는 1990년대 초반, **썬 마이크로시스템즈(Sun Microsystems)**의 **제임스 고스링(James Gosling)**과 그의 팀에 의해 탄생했다.

제임스 고스링은 당시 가전제품(TV, 셋톱박스 등) 안에 들어가는 소프트웨어를 개발하는 **'그린 프로젝트(Green Project)'**를 이끌고 있었다. 그는 다양한 종류의 하드웨어에서 공통적으로 실행될 수 있는 언어가 필요했고, 이를 위해 새로운 언어를 설계하게 된다.

초기에는 사무실 창밖에 있던 떡갈나무를 보고 **오크(Oak)**라는 이름을 붙였으나, 이미 상표로 등록된 이름이었기에 커피를 좋아하던 팀원들의 제안으로 **자바(Java)**라는 이름으로 변경되었다. 1995년 자바 1.0이 공식적으로 발표되면서 전 세계에 알려지게 되었다.

## WORA: Write Once, Run Anywhere
제임스 고스링이 자바를 설계할 때 가장 중요하게 여긴 철학은 **"한 번 작성하면 어디서든 실행된다(Write Once, Run Anywhere)"**는 것이었다.

당시에는 운영체제(OS)나 하드웨어가 달라지면 프로그램을 그에 맞춰 다시 작성하거나 컴파일해야 하는 불편함이 있었다. 자바는 **JVM(자바 가상 머신)**을 통해 이 문제를 해결했다. 개발자는 운영체제에 상관없이 자바 코드를 한 번만 작성하면, 어떤 환경에서든 동일하게 실행될 수 있게 된 것이다.

## 자바의 발전
자바는 1995년 발표 이후 폭발적인 인기를 얻으며 웹(Web) 프로그래밍의 핵심 언어로 자리 잡았다. 초기에는 웹 브라우저에서 실행되는 **애플릿(Applet)**으로 주목받았으나, 보안 및 성능 문제로 현재는 서버 사이드(Back-end) 개발과 안드로이드 앱 개발, 대규모 엔터프라이즈 시스템의 주력 언어로 사용되고 있다.

2010년 오라클(Oracle)이 썬 마이크로시스템즈를 인수하면서 자바의 소유권은 오라클로 넘어갔다. 현재 자바는 6개월마다 새로운 버전이 출시되며 빠르게 발전하고 있으며, 특히 LTS(Long Term Support) 버전인 Java 8, 11, 17, 21 등이 널리 사용되고 있다.
