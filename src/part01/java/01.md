# 1. 프로그래밍 언어와 자바

## 프로그래밍 언어의 필요성
컴퓨터는 우리가 일상생활에서 사용하는 언어를 직접 이해하지 못한다. 컴퓨터는 **기계어(Machine Language)**라고 불리는 0과 1로 이루어진 이진 코드(Binary Code)만을 이해하고 실행할 수 있다.

반면, 사람은 기계어를 이해하기 매우 어렵다. 그래서 사람과 컴퓨터가 대화하기 위해서는 **프로그래밍 언어(Programming Language)**라는 다리가 필요하다. 프로그래밍 언어는 사람이 이해하기 쉬운 문법으로 작성되며, 이를 컴퓨터가 이해할 수 있는 기계어로 번역해주는 역할을 한다.

## 고급 언어와 저급 언어
프로그래밍 언어는 컴퓨터 친화적이냐 사람 친화적이냐에 따라 **저급 언어**와 **고급 언어**로 나뉜다.

### 저급 언어 (Low-Level Language)
- 기계어에 가까운 언어로, 컴퓨터가 직접 이해하기 쉽다.
- 대표적으로 **어셈블리어(Assembly Language)**가 있다.
- 실행 속도가 빠르고 하드웨어를 직접 제어할 수 있지만, 배우기 어렵고 유지보수가 힘들다.

### 고급 언어 (High-Level Language)
- 사람이 이해하기 쉬운 문법을 사용한다.
- **자바(Java)**, **C**, **Python**, **JavaScript** 등이 대표적이다.
- 사람이 작성한 소스 코드를 컴퓨터가 이해할 수 있는 기계어로 변환하는 과정(**컴파일**)이 필요하다.
- 가독성이 좋고 생산성이 높아 대부분의 소프트웨어 개발에 사용된다.

## 컴파일(Compile)
고급 언어로 작성된 파일은 컴퓨터가 바로 실행할 수 없다. 이를 컴퓨터가 이해할 수 있는 0과 1로 된 기계어로 변환해주는 과정을 **컴파일(Compile)**이라고 한다.

1. 개발자가 **소스 파일(Source File)**을 작성한다. (예: `Hello.java`)
2. **컴파일러(Compiler)**가 소스 파일을 분석하여 기계어(또는 바이트코드)로 변환한다.
3. 컴퓨터가 변환된 파일을 실행한다.

자바는 이러한 고급 언어 중 하나로, 강력한 기능과 범용성을 갖춘 대표적인 프로그래밍 언어이다.
