# Chapter 09. 중첩 선언과 익명 객체

## 학습목표

클래스 내부에 선언되는 중첩 클래스와 익명 객체의 활용법을 알아봅니다.

## 목차

### [9.1 중첩 클래스](./01/)

객체지향 프로그램에서는 클래스 간에 서로 긴밀한 관계를 맺고 상호작용한다. 클래스가 여러 클래스와 관계를 맺는 경우에는 독립적으로 선언하는 것이 좋으나, 특정 클래스와만 관계를 맺을 경우에는 중첩 클래스로 선언하는 것이 유지보수에 도움이 되는 경우가 많다. 중첩 클래스...

### [9.2 인스턴스 멤버 클래스](./02/)

인스턴스 멤버 클래스는 클래스의 멤버로 선언된 인스턴스 클래스를 말한다. `public class A { class B { } }` 와 같이 작성된다. 접근 제한자에 따른 인스턴스 멤버 클래스의 접근 범위와 생성 방법에 대해 학습한다.

### [9.3 정적 멤버 클래스](./03/)

정적 멤버 클래스는 `static` 키워드와 함께 바깥 클래스의 멤버로 선언된 클래스를 말한다. `public class A { static class B { } }` 와 같이 선언된다. 정적 멤버 클래스의 특징과 접근 범위에 대해 알아본다.

### [9.4 로컬 클래스](./04/)

생성자 또는 메소드 내부에서 선언된 클래스를 로컬(`Local`) 클래스라고 한다. 로컬 클래스는 생성자와 메소드가 실행될 동안에만 객체를 생성할 수 있다. 로컬 변수를 사용할 때의 주의점과 자바 버전에 따른 차이를 확인한다.

### [9.5 바깥 멤버 접근](./05/)

중첩 클래스는 바깥 클래스와 긴밀한 관계를 맺으면서 바깥 클래스의 멤버(필드, 메소드)에 접근할 수 있다. 하지만 중첩 클래스가 어떻게 선언되었느냐에 따라 접근 제한이 있을 수 있다. 정적 멤버 클래스 내부에서는 바깥 클래스의 필드와 메소드를 사용할 때 제한이 따른다. 정...

### [9.6 중첩 인터페이스](./06/)

중첩 인터페이스는 클래스의 멤버로 선언된 인터페이스를 말한다. 인터페이스를 클래스 내부에 선언하는 이유는 해당 클래스와 긴밀한 관계를 맺는 구현 객체를 만들기 위해서이다. 안드로이드와 같은 UI 프로그램에서 이벤트를 처리할 목적으로 많이 활용된다.

### [9.7 익명 객체](./07/)

익명(`Anonymous`) 객체는 이름이 없는 객체를 말한다. 명시적으로 클래스를 선언하지 않기 때문에 쉽게 객체를 생성할 수 있다는 장점이 있다. 익명 객체는 필드값, 로컬 변수값, 매개변수값으로 주로 사용된다. 익명 객체는 클래스를 상속하거나 인터페이스를 구현해야만 생성...
