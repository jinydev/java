# 3.12 연산의 방향과 우선순위 (Operator Precedence)

우리가 수학 시간에 "더하기보다 곱하기를 먼저 계산해야 해!"라고 배운 것처럼, 컴퓨터 내부에 있는 계산기(ALU)도 수많은 연산 기호들을 만났을 때 **어떤 것부터 순서대로 처리해야 할지** 철저한 규칙을 지키며 일합니다.

이 규칙을 이해하면 아무리 길고 복잡한 코드라도 컴퓨터가 어떤 순서로 맛있는 결과물을 요리해 낼지 한눈에 파악할 수 있답니다! 👨‍🍳

---

## 1. 컴퓨터 주방(CPU ALU)의 주문 처리 순서 🍔

컴퓨터의 두뇌인 CPU 안에는 **ALU(산술논리연산장치)** 라는 수학과 논리를 담당하는 거대한 주방이 있습니다. 이 주방에는 매일 수천만 개의 연산 주문(영수증)이 쏟아집니다.

![CPU ALU 레스토랑 웹툰](./img/precedence_cpu_restaurant_webtoon.png)

웹툰을 보세요! 주방장(ALU)은 주문이 들어온 순서대로 아무거나 대충 요리하지 않습니다. 
주문서(연산자)마다 **VIP 등급(우선순위)** 이 적혀 있기 때문에, 등급이 높은 주문을 먼저 처리하고, 등급이 낮은 일반 주문은 뒤로 미뤄서 큐(Queue)라는 대기줄에서 기다리게 만듭니다.

### 🏆 전체 연산자 VIP 등급표 (피라미드)

아래 그림은 자바 주방장이 정해놓은 절대적인 서열 피라미드입니다. 꼭대기에 가까울수록 **먼저 계산(요리)** 됩니다!

![연산자 우선순위 피라미드](./img/operator_precedence_pyramid.svg)

*(💡 팁: 이 표를 억지로 다 외울 필요는 없습니다! 쓰다 보면 자연스럽게 손에 익습니다.)*

#### 예를 들어볼까요?
```java
boolean result = x > 0 && y < 0;
```
주방장에게 `>` (비교 연산), `<` (비교 연산), `&&` (논리 연산) 주문이 동시에 들어왔습니다.
피라미드를 보면 비교 연산(`>`,`<`)이 논리 연산(`&&`)보다 서열이 위에 있습니다.
그래서 주방장은 **1. `x > 0`을 먼저 계산**하고, **2. `y < 0`을 계산**한 뒤, **3. 그 두 결과를 가지고 `&&` 연산**을 마지막으로 요리합니다.

## 2. 같은 등급표를 가진 손님끼리 싸움이 났다면? (연산의 방향) 🤜🤛

만약 서열이 똑같은 기호들이 우르르 메뉴를 시켰다면 어떻게 할까요? 이때는 **'연산의 방향'** 이라는 교통정리 규칙을 따릅니다.

### 1) 대부분은 왼쪽 ➔ 오른쪽 (먼저 온 사람 먼저!)

대부분의 연산자(사칙연산, 비교, 논리 등)는 우리글을 읽는 것처럼 **왼쪽에서 오른쪽**으로 처리합니다.

```java
int result = 100 * 2 / 3 % 5;
```
`*`, `/`, `%` 는 곱셈/나눗셈 계열로 **우선순위 서열이 완벽히 똑같습니다.** 
그래서 주방장은 먼저 들어온 왼쪽 주문부터 차근차근 처리합니다.
1. `100 * 2` 를 요리해서 `200`을 만듦.
2. `200 / 3` 을 요리해서 몫인 `66`을 만듦.
3. 마지막으로 `66 % 5` 를 요리해서 최종 결과 `1`을 쟁반에 내보냄.

### 2) 예외! 오른쪽 ➔ 왼쪽으로 처리하는 청개구리 녀석들 🐸

딱 두 종류의 연산자는 특이하게 오른쪽부터 왼쪽으로 거꾸로 처리합니다: 바로 **단항 연산자(`++`, `--`, `!`)** 와 **대입 연산자(`=`, `+=`, `-=`)** 입니다.

```java
a = b = c = 5;
```
대입 연산자 `=` 는 오른쪽부터 처리됩니다!
1. `c` 상자에 `5`를 담습니다.
2. `b` 상자에 `c`의 내용물(`5`)을 담습니다.
3. `a` 상자에 `b`의 내용물(`5`)을 담습니다.
결국 세 변수 모두 5가 됩니다.

## 3. 🛡️ 에라, 모르겠다! 무적의 방패 "괄호 ()" 🛡️

막상 실무에서 코딩을 하다 보면 연산자가 4~5개씩 섞이게 되고, "이게 먼저 계산되던가.. 저게 먼저던가?" 헷갈리기 시작합니다. (전문 개발자들도 자주 헷갈립니다!)

**이럴 땐 고민하지 말고 그냥 괄호 `( )` 로 묶어버리세요!**
자바 주방장은 다른 모든 VIP 등급을 다 무시하고, **괄호 쳐진 주문서를 가장 최우선(0순위)으로 무조건 먼저 처리**합니다.

```java
int var1 = 1;
int var2 = 3;
int var3 = 2;

// 괄호가 없을 때: 곱셈이 덧셈보다 먼저 요리됨
int result1 = var1 + var2 * var3; // 1 + (3 * 2) = 7

// 괄호를 쳤을 때: 곱셈 서열을 무시하고 괄호부터 요리함!
int result2 = (var1 + var2) * var3; // (1 + 3) * 2 = 8
```

> **👨‍💻 선배 개발자의 조언:**
> "우선순위 규칙에 자신 있더라도, 다른 사람이 내 코드를 읽을 때 헷갈리지 않도록 웬만하면 의미 단위를 괄호 `()` 로 확실하게 묶어주는 것이 '아름다운 코드 퀄리티(가독성)'를 만드는 핵심 비결입니다!"
