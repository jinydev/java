# 1.4 바이트코드와 JVM

자바 프로그램이 실행되는 과정은 **"요리"**와 비슷합니다.

## 1. 실행 과정 3단계

### 1단계: 소스 코드 작성 (`.java`) 📝
*   개발자가 영어로 코드를 짭니다.
*   **비유**: 셰프가 **자필 레시피**를 적습니다.

### 2단계: 컴파일 (`.class`) 🖨️
*   `javac` (컴파일러)가 소스 코드를 **바이트코드**로 번역합니다.
*   바이트코드는 기계어가 아니라, **JVM이 이해하는 중간 언어**입니다.
*   **비유**: 자필 레시피를 보기 좋게 **인쇄된 표준 레시피**로 만듭니다.

### 3단계: 실행 (JVM) 🍳
*   `java` (JVM) 명령어가 바이트코드를 읽어서 컴퓨터(OS)에게 기계어로 통역해줍니다.
*   **비유**: 요리사(JVM)가 인쇄된 레시피를 보고 **실제로 요리**를 합니다.

```mermaid

    A[Hello.java<br>(소스 코드)] -->|javac<br>컴파일| B[Hello.class<br>(바이트코드)]

```

## 2. JVM (Java Virtual Machine)

**"자바를 실행하기 위한 가상의 컴퓨터"**입니다.
운영체제(Windows, Mac)는 서로 말이 다른데, JVM이 중간에서 **통역**을 해줍니다.

*   `Hello.class` 파일은 하나지만,
*   **Windows용 JVM**, **Mac용 JVM**이 각각 있어서 알아서 운영체제에 맞게 실행해줍니다.
*   그래서 개발자는 운영체제 상관없이 코드를 딱 한 번만 짜면 됩니다.
