# 3. 자바의 특징

자바가 오랫동안 사랑받는 이유는 강력하고 안정적인 특징들 덕분이다.

## 1. 이식성이 높다 (WORA)
자바의 가장 큰 특징은 **이식성(Portability)**이다. "Write Once, Run Anywhere"라는 슬로건처럼, 자바로 작성된 프로그램은 윈도우, 맥OS, 리눅스 등 운영체제를 가리지 않고 실행된다. 이는 **JVM(자바 가상 머신)**이 중간에서 번역기 역할을 해주기 때문이다.

## 2. 객체지향 언어 (Object-Oriented Programming, OOP)
자바는 100% 객체지향 언어이다. 프로그램을 수많은 **'객체(Object)'**라는 단위로 나누고, 이 객체들이 서로 상호작용하도록 설계하는 방식이다.
- **캡슐화, 상속, 다형성** 같은 객체지향의 특징을 완벽하게 지원한다.
- 유지보수가 쉽고, 대규모 프로젝트 개발에 적합하다.

## 3. 메모리를 자동으로 관리한다 (Garbage Collection)
C나 C++ 같은 언어는 개발자가 메모리를 직접 할당하고 해제해야 한다. 이는 강력하지만, 실수로 메모리를 해제하지 않으면 메모리 누수(Memory Leak)가 발생하여 프로그램이 다운될 수 있다.
반면, 자바는 **가비지 컬렉터(Garbage Collector, GC)**가 사용하지 않는 메모리를 자동으로 찾아 제거해준다. 개발자는 메모리 관리에 신경 쓰지 않고 핵심 로직 작성에만 집중할 수 있다.

## 4. 풍부한 오픈소스 라이브러리 (Open Source Ecosystem)
자바는 전 세계적으로 거대한 개발자 커뮤니티와 생태계를 가지고 있다. 수많은 검증된 오픈소스 라이브러리가 무료로 제공되므로, 개발자는 필요한 기능을 처음부터 만들 필요 없이 가져다 쓰기만 하면 된다. 이는 개발 시간을 획기적으로 단축시켜 준다.

## 5. 멀티 스레드(Multi-thread) 지원
자바는 운영체제의 도움 없이 자체적으로 멀티 스레드 프로그래밍을 쉽게 구현할 수 있는 API를 제공한다. 하나의 프로그램에서 동시에 여러 작업을 처리해야 하는 환경(대용량 데이터 처리, 웹 서버 등)에서 매우 유리하다.
